    Module 11 — Interactive Dashboard: Labour Market

    What We’re Building

    A three-chart dashboard where clicking a year on the top chart updates the two charts below — no custom JavaScript, no Turbo Frames. Turbo Morph handles the DOM update: only what changed re-renders.

    The parent chart is a line chart showing employment trends over time. Clicking any year updates two child charts: a grouped bar showing employment and unemployment by state, and a pie chart showing employment share by state — both for the selected year.

    The mechanism is adding a select_url value to the main chart. This action will be triggered when the user clicks on this chart.

    # app/services/stats/labour_force_snapshot.rb
module Stats
  module LabourForceSnapshot
    extend self

    def call(year:)
      LabourForceReading
        .where(year: year)
        .group(:state)
        .select(
          :state,
          "ROUND(AVG(employed_thousands), 1)   AS employed",
          "ROUND(AVG(unemployed_thousands), 1) AS unemployed",
          "ROUND(AVG(participation_rate), 1)   AS participation",
          "ROUND(AVG(unemployment_rate), 1)    AS rate"
        )
        .order(:state)
        .each_with_object({}) do |r, h|
          h[r.state] = {
            employed:      r.employed.to_f,
            unemployed:    r.unemployed.to_f,
            participation: r.participation.to_f,
            rate:          r.rate.to_f
          }
        end
    end
  end
end

    SQL does the aggregation — one query, no Ruby-side averaging. The result is a Hash keyed by state name, each value containing all four metrics.

    # app/views/components/charts/labour_force_timeline.rb
module Components
  module Charts
    class LabourForceTimeline < Components::Chart
      prop :readings, _Any, default: -> { [] }

      private

      def chart_options
        data = ::Stats::LabourForce.call(@readings)

        ::Chart::Options.new(
          color:   "tableau",
          tooltip: { trigger: "axis", formatter: "thousands" },
          legend:  { type: "scroll", bottom: 5 },
          x_axis:  {
            type:         "category",
            data:         years(data),
            triggerEvent: true   # enables click events on axis labels
          },
          y_axis:  {
            type:      "value",
            axisLabel: { formatter: "thousands" }
          },
          grid:    { left: 8, right: 8, bottom: 40, containLabel: true },
          series:  build_series(data)
        )
      end

      def build_series(data)
        data.map do |state, rows|
          ::Chart::Series::Line.new(
            name:   state,
            data:   rows.map { |r| r[:employed] },
            smooth: true,
            symbolSize: 6
          )
        end
      end

      def years(data)
        data.values.first&.map { |r| r[:year].to_s } || []
      end
    end
  end
end

    # app/views/components/charts/labour_force_share_pie.rb
module Components
  module Charts
    class LabourForceSharePie < Components::Chart
      prop :data, _Any,    default: -> { {} }
      prop :year, Integer, default: -> { Date.today.year }

      private

      def chart_options
        ::Chart::Options.new(
          color:   "tableau",
          tooltip: { trigger: "item", formatter: "stateSlice" },
          legend:  { type: "scroll", bottom: 5 },
          series: [
            ::Chart::Series::Pie.new(
              name:   "Employment Share #{@year}",
              data:   @data.map { |state, m|
                { name: state, value: m[:employed] }
              },
              radius: ["35%", "65%"],
              center: ["50%", "45%"],
              label:  { formatter: "{b}\n{d}%" }
            )
          ]
        )
      end
    end
  end
end

    Same "tableau" palette on all three components — the colour assigned to each state is consistent across all three charts. NSW is always the same colour whether it appears as a line, a bar, or a pie slice.

def labour_dashboard
  @year     = (params[:year] || LabourForceReading.maximum(:year)).to_i
  @readings = LabourForceReading.ordered
  @snapshot = Stats::LabourForceSnapshot.call(year: @year)

    11.5— What Turbo Morph Does

    When the user clicks “2020” on the timeline:

    1. chart_controller.js calls Turbo.visit("/charts/labour_dashboard?year=2020")
    2. Turbo fetches the new page
    3. Morph compares old and new HTML
    4. The timeline is identical — same options, same ECharts instance — Morph leaves it
    5. The bar and pie charts have new data — Morph replaces them
    6. ECharts reinitialises on the new mount divs
    7. The year heading updates from “2024” to “2020”

    No Turbo Frames. No broadcast. No custom stream. The page re-renders completely server-side — Morph surgically updates only what changed.

    11.6 — Colour Consistency

    All three components use color: "tableau". The service returns states in alphabetical order — ACT, NSW, NT, QLD, SA, TAS, VIC, WA. ECharts assigns palette colours in series order.

    The result: ACT is always the first tableau colour, NSW always the second — on the line chart, the bar chart, and the pie chart. A reader can identify a state by colour on any chart without reading the legend.

    This is the colour correspondence principle from Module 03 applied across an interactive dashboard.

    Infrastructure changes (see Appendix B and C):

    File Change
    chart_controller.js selectUrl value + #bindClick method
    app/views/components/chart.rb select_url: prop

    Patterns introduced:

    • select_url: prop — opt-in click navigation on any chart
    • triggerEvent: true on x axis — enables click events on axis labels
    • Turbo.visit with action: "replace" — updates URL without history entry
    • Turbo Morph — only changed charts re-render, parent chart untouched
    • params[:year] — controller reads selected year, fetches snapshot data
    • Colour correspondence across chart types — same palette, same series order

    The interaction principle:

    The parent chart navigates. The controller fetches. Morph updates. No custom JavaScript per chart. No Turbo Frames. No broadcasts. Click a data point — the page responds.
